Executive Summary
Audit Overview & Critical Findings
JHB Curtain Cleaning (jhbcurtaincleaning.co.za) has a solid technical foundation built on Next.js 16 with Static Site Generation (SSG), well-structured JSON-LD schema on the homepage and landing pages, and good core page speed metrics. The site's 18 landing pages — spanning services, sectors, and Johannesburg service areas — provide excellent topical authority potential and represent a thoughtful content architecture that few competitors in this niche can match.
However, the audit reveals a catastrophic NAP data corruption issue that undermines virtually every SEO and local visibility signal. The entire codebase has reverted to the old brand identity: "On The Spot Curtain Cleaning Specialists" with the old domain (curtaincleaning.org), old address (Martha North Rd, Randburg), wrong founder (Kathy Dunlop), and wrong contact details. This data appears in over 70 instances across site-data.ts, layout.tsx, JSON-LD schemas, meta tags, FAQ content, and component files. Google's Knowledge Graph may be creating duplicate or conflicting entities as a result, effectively destroying local ranking authority.
Beyond the NAP crisis, the site is missing critical SEO infrastructure: no XML sitemap, no canonical tags, no blog/content hub for AEO optimization, no Organization schema, and no hreflang or locale targeting. Image assets are nearly non-existent (only 2 logo PNGs on the entire site), and there are no before/after photos — the single most important trust signal for a cleaning service. Internal linking between the 18 landing pages is absent in body content, preventing link equity flow and semantic context from connecting related services, sectors, and areas.
The 18 landing pages provide a strong topical cluster foundation, but they need AEO optimization (direct-answer formatting, conversational keyword targeting, pricing transparency) and cross-linking to realize their potential. With immediate NAP correction and the implementation roadmap outlined in this report, the site can realistically achieve a score of 75+ within 60 days.

Task 1 /ai-seo
AI Search Engine Optimization Strategy
Critical  Entity Identity Crisis
The JSON-LD schema in layout.tsx still references "On The Spot Curtain Cleaning Specialists" with URL "curtaincleaning.org" and address "Martha North Road, Randburg." AI systems cannot confidently match this business to real-world entities. The @id field still points to curtaincleaning.org. Google's Knowledge Graph may create duplicate or conflicting entities.
Critical  No Organization Schema
The site lacks a top-level Organization schema. AI crawlers use this to understand corporate entity, founding, and authority. Without it, the business has no verified organizational identity in knowledge graphs.
High  No BreadcrumbList Schema
The 18 landing pages have no breadcrumb markup. AI systems use breadcrumbs to understand page hierarchy and topical relationships.
High  Insufficient Direct Answers
FAQ content is buried inside accordion components. While text is crawlable, AI Overviews prefer content that answers questions in the first 1-2 sentences before expanding. Current FAQ answers are long-form paragraphs.
Medium  No Speakable Specification
No speakable schema markup exists, which tells voice assistants and AI which content is suitable for audio playback.
Implementation Plan
  1. Fix NAP data across ALL files immediately. JSON-LD @id, name, url, telephone, email, and address must match canonical NAP format exactly.
  2. Add Organization schema to layout.tsx alongside the existing LocalBusiness with @id, name, url, logo, sameAs, founder (Stephen Dunlop), and foundingDate (2009).
  3. Add BreadcrumbList schema to every landing page with Home / Services (or Sectors/Areas) / Page Name hierarchy.
  4. Restructure FAQ answers to lead with a direct 1-2 sentence answer, then expand. Example: "Can you clean curtains without taking them down? Yes. We clean curtains exactly where they hang using on-site dry-cleaning equipment."
  5. Add speakable schema to FAQ items for voice search optimization.

Task 3 /seo-audit
Technical SEO & Site Health
Critical  No XML Sitemap
There is no sitemap.xml or app/sitemap.ts file. Search engines have no structured map of the 19 pages. This directly impacts crawl efficiency and indexation speed.
Critical  No Canonical Tags
No canonical URL tags exist on any page. Without canonicals, search engines may treat www/non-www, http/https, and trailing-slash variants as duplicate content.
Medium  next.config.ts Issues
typescript: { ignoreBuildErrors: true } masks type errors. reactStrictMode: false disables React strict mode. No image optimization configuration (formats, sizes).
Medium  No Custom 404 or Error Boundary
No custom not-found.tsx exists. No error.tsx at the app level. Runtime errors show a blank page.
Good  Font Loading & Mobile Responsiveness
Inter and Playfair Display use display: 'swap' — eliminates FOIT. Tailwind responsive prefixes used throughout. All grids collapse to single column on mobile.
Medium  Lang Attribute Wrong
<html lang="en"> should be <html lang="en-ZA"> for South African English locale targeting.
High  Page Speed Concerns
The 844-line contact-section.tsx and 605-line header.tsx are large client components. Framer Motion adds ~45KB to the bundle. No code splitting optimization visible.
Implementation Plan
  1. Create src/app/sitemap.ts with all static pages, service pages, sector pages, and area pages. Include lastModified, changeFrequency, and priority values.
  2. Add canonical URLs to layout.tsx metadata: metadataBase, alternates: { canonical: '/' }. And to each landing page generateMetadata.
  3. Fix html lang attribute from "en" to "en-ZA".
  4. Create custom not-found.tsx and error.tsx pages.
  5. Update next.config.ts — remove ignoreBuildErrors, enable reactStrictMode, add image config with formats: ['image/avif', 'image/webp'].
import { MetadataRoute } from 'next' const BASE_URL = 'https://www.jhbcurtaincleaning.co.za' export default function sitemap(): MetadataRoute.Sitemap { const staticPages = [ { url: BASE_URL, lastModified: new Date(), changeFrequency: 'weekly', priority: 1.0 } ] // ...service, sector, and area pages return [...staticPages, ...servicePages, ...sectorPages, ...areaPages] }
